# PM-Worker Document

Agent tool 병렬 실행으로 대규모 문서를 효율적으로 작성합니다.

## 활성화 시점

- 대규모 설계 문서 작성 (500줄+)
- 종합 분석 보고서 (여러 파일 분석 필요)
- 기술 명세서 작성
- 프로젝트 감사 보고서

## 3 Phase 워크플로우

### Phase 1: PM — 구조 설계

1. **문서 목적/대상 독자 정의**
2. **섹션 분할** (3~5개, Worker 수에 맞춤)
3. **Worker별 담당 영역 배정**
4. **통합 규칙 정의**:
   - 용어 사전 (일관된 명칭)
   - 표 형식 (Markdown 테이블 규격)
   - 교차 참조 형식
   - 분석 출력 형식 (파일명/줄수/주제/요약/인사이트)

### Phase 2: Workers — 병렬 작성

Agent tool로 Worker를 `run_in_background: true`로 병렬 실행:

```
PM이 Agent tool 호출 (동시에 3~5개):

Agent(
  description: "섹션 A 분석",
  prompt: "다음 파일들을 읽고 분석해주세요: [파일 목록]
    각 파일에 대해 다음을 정리:
    - 파일명, 줄 수
    - 핵심 주제/목적
    - 주요 내용 요약 (3-5줄)
    - 다른 문서와의 관계
    - [목적]에 활용 가능한 인사이트
    마지막에 종합 요약 추가.",
  subagent_type: "general-purpose",
  run_in_background: true,
  name: "worker-1-section-a"
)
```

**Worker 프롬프트 핵심 요소:**
- 분석 대상 파일 목록 (절대 경로)
- 출력 형식 명확 지정
- 종합 요약 요청
- 읽기 전용 (코드 변경 X)

### Phase 3: PM — 통합

Worker 완료 후 PM이 수행:

1. **중복 제거**: 여러 Worker가 같은 내용을 다룬 경우
2. **일관성 검증**: 용어, 수치, 참조 교차 확인
3. **교차 참조 링크**: 섹션 간 참조 연결
4. **Executive Summary**: 전체 핵심 발견 요약
5. **최종 품질 검증**: 누락/모순 확인

## 실전 예시

이 프로젝트에서 수행한 100개 MD 분석:

| Worker | 담당 | 파일 수 | 소요 |
|--------|------|--------|------|
| W1 | 설계/기획 문서 | 19 | ~4분 |
| W2 | 구현/개발 문서 | 31 | ~4.5분 |
| W3 | Wiki 콘텐츠 | 21 | ~3분 |
| W4 | 개발계획/마일스톤 | 29 | ~5분 |

→ PM 통합: `docs/HCHAT_WIKI_SKILL_DESIGN.md` (725줄)

## Worker 배분 가이드

| 문서 규모 | Worker 수 | 권장 분할 |
|---------|----------|---------|
| 10-20개 파일 | 2 | 주제별 |
| 20-50개 파일 | 3-4 | 디렉토리별 |
| 50-100개 파일 | 4-5 | 영역별 |
| 100개+ | 5+ | 영역 + 하위 분류 |

## 주의사항

- Worker는 **읽기 전용**: Write 권한 이슈 발생 가능 → PM이 직접 파일 생성
- Worker 간 **파일 중복 금지**: 동일 파일을 여러 Worker에 배정하지 않음
- **완료 대기**: `run_in_background: true`로 실행 후 자동 알림 대기

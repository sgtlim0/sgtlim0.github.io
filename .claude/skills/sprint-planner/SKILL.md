# Sprint Planner

에픽→스토리→태스크 3단 분해 + Day별 계획 + 리스크 관리 통합 스킬.

## 활성화 시점

- 새 스프린트 시작
- 기능 구현 계획 수립
- 대규모 리팩토링 계획
- 마일스톤 일정 수립

## 입력

- **에픽 목록**: 구현할 기능/작업 리스트
- **팀 인력**: FE/BE/Infra 역할별 인원
- **기간**: 스프린트 기간 (일/주)

## 산출물 8개 섹션

### 1. 에픽 분해

```markdown
## Epic 1: {에픽명}

### Story 1.1: {스토리}
- [ ] Task 1.1.1: {태스크} (FE, 3SP)
- [ ] Task 1.1.2: {태스크} (BE, 2SP)
```

### 2. Day별 계획

| Day | 태스크 | 담당 | 의존성 | 상태 |
|-----|--------|------|--------|------|
| D1 | T01 인프라 설정 | BE | 없음 | |
| D1 | T02 UI 스캐폴딩 | FE | 없음 | |
| D2 | T03 API 연동 | BE | T01 | |

### 3. 의존성 그래프

```mermaid
gantt
    title Sprint N
    section FE
    UI 스캐폴딩: t02, d1, 1d
    컴포넌트 구현: t04, after t02, 2d
    section BE
    인프라 설정: t01, d1, 1d
    API 구현: t03, after t01, 2d
```

### 4. 인력 매핑

| 역할 | 담당자 | D1 | D2 | D3 | D4 | D5 |
|------|--------|----|----|----|----|-----|
| FE | - | T02 | T04 | T04 | T06 | T08 |
| BE | - | T01 | T03 | T03 | T05 | T07 |

### 5. Go/No-Go 체크리스트

**Must (4개, 모두 충족 시 Go):**
- [ ] 핵심 기능 동작 확인
- [ ] 크리티컬 버그 0건
- [ ] 기본 테스트 통과
- [ ] 데모 시나리오 실행 가능

**Should (4개, 2개 이상 권장):**
- [ ] 커버리지 임계값 충족
- [ ] 성능 기준 충족
- [ ] 코드 리뷰 완료
- [ ] 문서 업데이트

### 6. Scope 축소 전략

| Level | 조건 | 대응 |
|-------|------|------|
| **L1** | 1-2일 지연 | Nice-to-have 기능 제거 |
| **L2** | 3-4일 지연 | Should 기능 다음 스프린트로 이동 |
| **L3** | 5일+ 지연 | Must 기능만 남기고 스프린트 축소 |

### 7. 리스크 레지스터

| ID | 리스크 | 확률 | 영향 | Recovery Action |
|----|--------|------|------|----------------|
| R1 | API 지연 | 중 | 높 | Mock 데이터로 FE 진행 |
| R2 | 기술 난이도 | 낮 | 높 | PoC 우선 진행 |

### 8. 데모 시나리오

**중간 데모 (Day N/2):**
- 핵심 흐름 1가지 시연 (3분)
- 블로커 공유

**최종 데모 (Day N):**
- 전체 기능 시연 (5분)
- 메트릭 공유

## PM-Worker 병렬 작성 지원

복잡한 Sprint에서는 Agent tool로 Worker를 병렬 배치:
- Worker A: FE 태스크 상세화
- Worker B: BE 태스크 상세화
- Worker C: 테스트/인프라 태스크
- PM: 통합 + 의존성 그래프 + 리스크

### 2.2 비전: "브라우저가 곧 업무 환경"

> 임직원 업무 시간의 90% 이상이 Chrome 브라우저에서 이루어집니다.
> H Chat은 Chrome Side Panel 단일 접점에서 모든 시스템을 AI로 통합합니다.

**미션**: 브라우저를 떠나지 않고, Side Panel AI와 자연어로 대화하며 정보 탐색 → 분석 → 의사결정 → 실행

**UI 서피스 4종**:

|--------|------|

### 3.8 Extension 개발 핵심 주의사항 4가지

| 이슈 | 설명 | 대응 |
|------|------|------|
| **DOM Race Condition** | SPA/Shadow DOM에서 waitForDOMStable() 필수 | MutationObserver로 변경 감지 후 100ms 안정화 대기. SAP Fiori, Jira Cloud 등 SPA 사이트 대응 |
| **API 키 Two-Phase 삭제** | Extension 제거 시 API 키 즉시 무효화 불가 | Phase 1: onSuspend에서 서버 무효화 요청. Phase 2: 서버 측 24h TTL 만료로 이중 안전장치 |
| **Reflow 최소화** | getBoundingClientRect() 호출 배치 처리 필수 | Content Script에서 DOM 읽기/쓰기 분리. IntersectionObserver 우선 사용 → Layout Thrashing 방지 |
| **SSE AbortController** | 스트리밍 중 취소/탭 전환 시 AbortController.abort() 필수 | SW↔Side Panel SSE 연결 라이프사이클 명시 관리. 미처리 시 메모리 누수/좀비 연결 발생 |

### 3.9 Chrome Extension 차별화 vs 앱/PWA

| 비교 요소 | 기존 접근 (앱/PWA) | Chrome Extension (H Chat) |
|---------|----------------|--------------------------|
| 설치 | 별도 다운로드 / URL 접속 | **관리자 강제 배포 (GPO) — 클릭 불필요** |
| 컨텍스트 | 앱 전환 필요, 복사/붙여넣기 | **현재 페이지 DOM 직접 읽기** |
| 접근성 | 별도 창/탭 필요 | **모든 탭에서 즉시 호출** |
| 관리 | MDM 별도 구성 | **Chrome 정책으로 통합 관리** |
| 업데이트 | 수동 / 스토어 업데이트 | **CWS 자동 업데이트 — 무중단** |
| 제거 방지 | 불가 | **ExtensionInstallForcelist 정책** |

> 데이터 플라이휠: 사용 증가 → 개인화 → 의존도↑ → Switching Cost 기하급수적 증가 → 경쟁사 진입 차단

## 7. 심층 분석 의견

### 7.1 강점 평가

1. **명확한 문제-해법 적합성**: 7+ 시스템 분절 문제를 Chrome Extension의 컨텍스트 네이티브 접근으로 정확히 해결. "별도 앱 전환 불필요"라는 UX 원칙이 모든 설계에 일관되게 반영됨

2. **현실적 재무 모델**: LTV/CAC 48x, Payback < 1개월은 B2B SaaS 벤치마크 상위. ARPU $13→$20 상승 경로가 Enterprise 구독 비중 증가와 맞물려 자연스러움

3. **4중 해자 전략의 시간차 설계**: L1(데이터 주권, 6개월) → L2(사내 통합, 12개월) → L3(에이전트 생태계, 18개월+)로 복제 난이도가 시간에 비례하여 증가하는 구조

4. **보안 아키텍처의 깊이**: 6계층 Zero Trust + PII 11패턴 + 블록리스트 20+6 패턴은 엔터프라이즈 보안 심사를 통과할 수준

### 7.2 리스크/개선 의견

1. **SPA/Shadow DOM 리스크 과소평가 가능성**: SAP Fiori, Jira Cloud 등 SPA 사이트의 DOM 접근은 "높음/높음"으로 평가했으나, 실제로는 Chrome Extension MV3의 Content Script 제약이 더 심할 수 있음. Sprint 0 전 PoC가 올바른 판단

2. **$0.12/작업 비용 지속 가능성**: LLM API 가격이 현재 급격히 하락 중이나, 향후 안정화 시 비용 구조 변동 가능. 캐싱 60%↓ 전략이 핵심

3. **Forcelist 의존도**: 설치율 95%+ 목표가 Forcelist 정책에 전적으로 의존. Chrome 정책 변경이나 Edge 이전 시나리오에 대한 대비 필요

4. **13명 팀으로 30주 315SP 소화**: 인당 24.2SP(30주 기준). 스프린트당 21SP/팀은 현실적이나, 보안 심사와 병행 시 일정 리스크 존재

5. **hchat-pwa 관계 미정의**: 기존 PWA 모노레포(현재 프로젝트)와의 관계가 "협의" 상태. 이 결정이 아키텍처 전체에 파급 효과가 크므로 Sprint 0 전 반드시 확정 필요

### 7.3 결론

H Chat AI Browser OS는 **"브라우저가 곧 OS"** 라는 명확한 비전 하에, Chrome Extension을 엔터프라이즈 AI 에이전트의 실행 환경으로 정의한 전략적으로 탄탄한 기획서입니다. 특히 Forcelist 자동 배포 + Zero Trust 보안 + MARS 자율 에이전트의 3축 결합은 경쟁사가 단기간에 복제하기 어려운 구조적 해자를 형성합니다.

**투자 대비 기대 수익** ($645K 투자 → $12.1M ARR / 270% ROI)이 합리적이며, **30주 5단계 로드맵**이 각 마일스톤별 Go/No-Go 기준으로 리스크를 관리하는 구조는 실행 가능성을 높입니다.

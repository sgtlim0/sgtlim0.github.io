# 프롬프트 라이브러리

H Chat v2에서 도입된 프롬프트 라이브러리는 자주 사용하는 프롬프트를 템플릿으로 저장하고 빠르게 재사용할 수 있는 기능입니다.

## 프롬프트 라이브러리란?

프롬프트 라이브러리는 미리 정의된 프롬프트 템플릿을 저장하고 관리하는 시스템입니다. 반복적인 작업을 빠르게 수행하거나, 복잡한 프롬프트를 쉽게 재사용할 수 있습니다.

### 주요 기능

- **8개 기본 프롬프트**: 일반적인 작업을 위한 사전 정의 템플릿
- **커스텀 프롬프트**: 사용자 정의 프롬프트 생성/편집/삭제
- **단축어 지원**: `/` 로 시작하는 빠른 실행 명령어
- **변수 치환**: `{{변수명}}` 형식으로 동적 내용 삽입
- **카테고리 분류**: 프롬프트를 카테고리별로 구성

## 8가지 기본 프롬프트

H Chat은 일반적인 작업을 위한 8가지 기본 프롬프트를 제공합니다.

### 1. 요약 (Summarize)

**단축어**: `/sum` 또는 `/summarize`

**용도**: 긴 텍스트나 문서를 간결하게 요약

**템플릿**:
```
다음 내용을 3-5개의 핵심 포인트로 요약해주세요:

{{content}}
```

**사용 예시**:
- 뉴스 기사 요약
- 회의록 요점 정리
- 논문 초록 작성

### 2. 번역 (Translate)

**단축어**: `/tr` 또는 `/translate`

**용도**: 한국어↔영어 번역

**템플릿**:
```
다음 텍스트를 {{target_lang}}로 번역해주세요. 원문의 톤과 뉘앙스를 유지하세요:

{{content}}
```

**사용 예시**:
- 비즈니스 이메일 번역
- 기술 문서 번역
- 웹페이지 내용 번역

### 3. 교정 (Proofread)

**단축어**: `/proof` 또는 `/proofread`

**용도**: 맞춤법, 문법, 표현 교정

**템플릿**:
```
다음 텍스트의 맞춤법, 문법, 표현을 교정하고 개선점을 설명해주세요:

{{content}}
```

**사용 예시**:
- 보고서 교정
- 이메일 검토
- 블로그 글 다듬기

### 4. 코드 리뷰 (Code Review)

**단축어**: `/cr` 또는 `/codereview`

**용도**: 코드 품질, 보안, 성능 검토

**템플릿**:
```
다음 코드를 리뷰하고 개선점을 제시해주세요:
- 버그나 잠재적 오류
- 성능 개선 방안
- 보안 취약점
- 코드 가독성

{{code}}
```

**사용 예시**:
- Pull Request 리뷰
- 레거시 코드 분석
- 보안 취약점 검사

### 5. 이메일 작성 (Email)

**단축어**: `/email`

**용도**: 비즈니스 이메일 초안 작성

**템플릿**:
```
다음 정보로 전문적인 비즈니스 이메일을 작성해주세요:

목적: {{purpose}}
수신자: {{recipient}}
주요 내용: {{content}}
```

**사용 예시**:
- 거래처 이메일
- 회의 요청
- 제안서 발송

### 6. 설명 (Explain)

**단축어**: `/ex` 또는 `/explain`

**용도**: 복잡한 개념을 쉽게 설명

**템플릿**:
```
다음 개념을 {{level}} 수준으로 설명해주세요. 비유와 예시를 포함하세요:

{{concept}}
```

**사용 예시**:
- 기술 개념 학습
- 초보자를 위한 설명
- 발표 자료 준비

### 7. 브레인스토밍 (Brainstorm)

**단축어**: `/brain` 또는 `/brainstorm`

**용도**: 아이디어 발상, 문제 해결

**템플릿**:
```
다음 주제에 대해 창의적인 아이디어를 10가지 제시해주세요:

주제: {{topic}}
목표: {{goal}}
제약사항: {{constraints}}
```

**사용 예시**:
- 프로젝트 기획
- 마케팅 캠페인
- 문제 해결 방안 모색

### 8. 개선 (Improve)

**단축어**: `/imp` 또는 `/improve`

**용도**: 텍스트 품질 향상

**템플릿**:
```
다음 텍스트를 더 {{style}}하게 개선해주세요:

{{content}}
```

**사용 예시**:
- 글 퀄리티 향상
- 전문적인 표현으로 변환
- 간결하게 다듬기

## 커스텀 프롬프트 생성

사용자만의 프롬프트를 생성하여 라이브러리에 추가할 수 있습니다.

### 생성 방법

```markdown
1. 사이드패널에서 "프롬프트 라이브러리" 탭 열기
2. "+ 새 프롬프트" 버튼 클릭
3. 프롬프트 정보 입력:
   - 이름: 프롬프트 식별 이름
   - 카테고리: 분류 (선택)
   - 단축어: `/` 명령어 (선택)
   - 템플릿: 프롬프트 내용 (변수 사용 가능)
4. "저장" 클릭
```

### 커스텀 프롬프트 예시

#### 예시 1: 회의록 작성

```yaml
이름: 회의록 작성
단축어: /meeting
카테고리: 업무
템플릿: |
  다음 회의 내용을 정리하여 공식 회의록을 작성해주세요:

  회의명: {{meeting_name}}
  일시: {{date}}
  참석자: {{attendees}}
  안건: {{agenda}}

  다음 형식으로 작성:
  1. 회의 개요
  2. 논의 내용
  3. 결정 사항
  4. 액션 아이템 (담당자, 기한 포함)
```

#### 예시 2: 기술 블로그 작성

```yaml
이름: 기술 블로그
단축어: /blog
카테고리: 글쓰기
템플릿: |
  다음 주제로 개발자를 위한 기술 블로그 글을 작성해주세요:

  주제: {{topic}}
  대상 독자: {{audience}}
  핵심 메시지: {{key_message}}

  구조:
  - 도입 (문제 정의)
  - 솔루션 제시
  - 코드 예시 (실제 동작하는 코드)
  - 장단점 분석
  - 결론 및 다음 단계
```

#### 예시 3: SQL 쿼리 생성

```yaml
이름: SQL 쿼리
단축어: /sql
카테고리: 개발
템플릿: |
  다음 요구사항에 맞는 SQL 쿼리를 작성해주세요:

  데이터베이스: {{db_type}}
  요구사항: {{requirement}}
  테이블 구조: {{schema}}

  결과:
  1. 최적화된 쿼리
  2. 인덱스 제안
  3. 쿼리 설명
  4. 예상 결과 예시
```

## 단축어 (Slash Commands)

프롬프트 라이브러리의 가장 강력한 기능은 `/` 단축어입니다.

### 단축어 사용법

```markdown
1. ChatView 입력창에 `/` 입력
2. 자동완성 드롭다운이 표시됨
3. 화살표 키로 선택 또는 계속 타이핑하여 필터링
4. Enter로 프롬프트 선택
5. 변수가 있는 경우 자동으로 커서 이동
```

### 단축어 네이밍 규칙

| 규칙 | 예시 | 설명 |
|-----|------|------|
| 짧게 | `/sum`, `/tr`, `/cr` | 빠른 입력 |
| 명확하게 | `/email`, `/blog` | 용도 파악 용이 |
| 일관성 | `/code-*` 시리즈 | 관련 프롬프트 그룹화 |
| 충돌 방지 | `/my-custom` | 기본 프롬프트와 구분 |

### 단축어 자동완성

자동완성 드롭다운은 다음 정보를 표시합니다:

```
/sum → 요약 (Summarize)
       긴 텍스트를 간결하게 요약

/tr → 번역 (Translate)
      한영/영한 번역

/email → 이메일 작성
         비즈니스 이메일 초안
```

## 변수 치환 (Variable Substitution)

프롬프트 템플릿에 `{{변수명}}` 형식으로 변수를 정의하면, 실행 시 자동으로 값을 입력할 수 있습니다.

### 변수 문법

```
{{변수명}}           # 기본 변수
{{변수명:기본값}}    # 기본값 포함
{{변수명|선택1|선택2}} # 드롭다운 선택
```

### 변수 동작 방식

1. **프롬프트 선택**: 단축어 또는 라이브러리에서 선택
2. **변수 감지**: 템플릿에서 `{{...}}` 패턴 추출
3. **입력 요청**: 각 변수에 대해 입력 프롬프트 표시
4. **치환 실행**: 입력된 값으로 템플릿 완성
5. **전송**: 완성된 프롬프트를 AI에 전송

### 변수 사용 예시

#### 예시 1: 간단한 변수

**템플릿**:
```
{{content}}를 {{target_lang}}로 번역해주세요.
```

**실행 시**:
1. `content` 입력: "Hello World"
2. `target_lang` 입력: "한국어"

**결과**:
```
Hello World를 한국어로 번역해주세요.
```

#### 예시 2: 기본값 포함

**템플릿**:
```
{{text}}를 {{style:전문적}}인 톤으로 개선해주세요.
```

**실행 시**:
- `text` 입력: "이 제품 좋네요"
- `style` 입력: (Enter만 누르면 "전문적" 사용)

**결과**:
```
이 제품 좋네요를 전문적인 톤으로 개선해주세요.
```

#### 예시 3: 드롭다운 선택

**템플릿**:
```
{{code}}를 {{lang|Python|JavaScript|TypeScript|Go}}로 변환해주세요.
```

**실행 시**:
- `code` 입력: "def hello(): print('hi')"
- `lang` 선택: 드롭다운에서 "JavaScript" 선택

**결과**:
```
def hello(): print('hi')를 JavaScript로 변환해주세요.
```

## 카테고리 시스템

프롬프트를 카테고리별로 정리하여 관리할 수 있습니다.

### 권장 카테고리

| 카테고리 | 용도 | 프롬프트 예시 |
|---------|------|-------------|
| **업무** | 비즈니스 작업 | 이메일, 회의록, 보고서 |
| **개발** | 코딩 관련 | 코드 리뷰, 디버깅, SQL |
| **글쓰기** | 콘텐츠 작성 | 블로그, 교정, 요약 |
| **학습** | 교육 자료 | 설명, 튜토리얼, 퀴즈 |
| **번역** | 언어 변환 | 한영, 영한, 다국어 |
| **분석** | 데이터 분석 | 인사이트, 리포트, 차트 |

### 카테고리 필터링

```markdown
1. 프롬프트 라이브러리 탭에서 카테고리 드롭다운 선택
2. 원하는 카테고리 클릭
3. 해당 카테고리의 프롬프트만 표시
4. "전체" 선택으로 필터 해제
```

## 프롬프트 편집/삭제

### 편집 방법

```markdown
1. 프롬프트 라이브러리에서 편집할 프롬프트 선택
2. 우측 "편집" 버튼(✏️) 클릭
3. 정보 수정
4. "저장" 클릭
```

**주의**: 기본 8개 프롬프트는 편집 불가 (복제 후 수정 가능)

### 삭제 방법

```markdown
1. 프롬프트 우측 "삭제" 버튼(🗑️) 클릭
2. 확인 다이얼로그에서 "삭제" 클릭
3. 프롬프트가 영구 삭제됨
```

**주의**: 기본 프롬프트는 삭제 불가

## 프롬프트 공유

### 내보내기

```markdown
1. 프롬프트 선택
2. "내보내기" 버튼 클릭
3. JSON 파일 다운로드
```

### 가져오기

```markdown
1. "가져오기" 버튼 클릭
2. JSON 파일 선택
3. 프롬프트가 라이브러리에 추가됨
```

### JSON 구조

```json
{
  "id": "prompt-uuid",
  "name": "회의록 작성",
  "category": "업무",
  "shortcut": "/meeting",
  "template": "다음 회의 내용을 정리하여...",
  "variables": ["meeting_name", "date", "attendees"],
  "createdAt": 1709294400000
}
```

## 고급 활용 팁

### 팁 1: 페이지 컨텍스트와 결합

```
현재 페이지 내용을 {{style}} 스타일로 재작성해주세요.
```

페이지 컨텍스트가 활성화되어 있으면 자동으로 현재 페이지 내용이 포함됩니다.

### 팁 2: 에이전트 모드 프롬프트

```
다음 웹사이트 {{url}}을 분석하고 SEO 개선점을 찾아주세요.
```

에이전트 모드가 활성화되어 있으면 자동으로 웹사이트를 방문하여 분석합니다.

### 팁 3: 멀티 스텝 프롬프트

```
1단계: {{topic}}에 대해 조사
2단계: 핵심 포인트 5가지 추출
3단계: 각 포인트를 200자로 설명
4단계: 블로그 글 형식으로 정리
```

복잡한 작업을 단계별로 나누어 프롬프트에 포함합니다.

```
src/lib/promptLibrary.ts          # 프롬프트 CRUD

```

## 참고

- 프롬프트는 로컬 스토리지(`chrome.storage.local`)에 저장됩니다.
- 변수 치환은 실시간으로 처리되며, 입력 이력은 저장되지 않습니다.
- 단축어는 대소문자를 구분하지 않습니다 (`/SUM` = `/sum`).
- 프롬프트 라이브러리는 무제한 개수 생성 가능합니다.

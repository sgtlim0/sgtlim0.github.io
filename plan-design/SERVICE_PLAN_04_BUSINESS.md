### 2.2 Chrome Extension 배포 모델

| 배포 방식 | 대상 | 설명 |
|-----------|------|------|
| **Chrome Web Store (사내 비공개)** | 전체 | Google Workspace 조직 내 비공개 게시, 외부 비노출 |
| **Google Admin Console 관리 배포** | Pro 이상 | 조직 단위(OU)별 Extension 자동 설치/업데이트 관리 |
| **ExtensionInstallForcelist (강제 설치)** | Enterprise | Group Policy / Chrome Browser Cloud Management로 무설치 배포, 사용자 제거 불가 |
| **Allowlist/Blocklist 정책** | Enterprise | 허용된 사이트에서만 Extension 활성화, 민감 사이트 자동 비활성화 |

## 3. Go-to-Market 전략

### 3.1 3단계 GTM (Chrome Extension 중심)

```
Stage 1: 내부 파일럿       Stage 2: 그룹사 확산       Stage 3: 외부 B2B
(0~6개월)                  (6~18개월)                 (18~30개월)
┌─────────────────┐      ┌─────────────────┐      ┌─────────────────┐
│ Chrome Web Store │      │ Admin Console    │      │ 외부 기업         │
│ 사내 비공개 배포  │ ──→ │ 강제 설치 배포    │ ──→ │ Chrome 관리 정책   │
│ 오토에버 500명    │      │ 16개 계열사       │      │ 제조/금융/물류     │
│ PMF 검증         │      │ 25,000명 확대     │      │ 50,000+ 좌석      │
└─────────────────┘      └─────────────────┘      └─────────────────┘
```

### 3.2 단계별 GTM 상세

**Stage 1: 내부 파일럿 (0~6개월) -- Chrome Web Store 사내 비공개 배포**

| 항목 | 내용 |
|------|------|
| **타겟** | 현대오토에버 IT/경영기획 부서 500명 |
| **배포 방식** | Chrome Web Store 사내 비공개 게시 -> 챔피언 사용자 수동 설치 |
| **목표** | PMF 검증, NPS 40+, DAU 60%+, Extension 활성 사용률 80%+ |
| **전략** | 챔피언 사용자 20명 선발 -> 피드백 루프 -> 빠른 반복, Chrome Web Store 평점 4.0+ 달성 |
| **성공 기준** | 주간 리텐션 70%, 자동화율 30%+, 경영진 승인, Extension 설치 유지율 90%+ |

**Stage 2: 그룹사 확산 (6~18개월) -- Google Admin Console 강제 설치**

| 항목 | 내용 |
|------|------|
| **타겟** | 현대차, 기아, 현대모비스 등 16개 계열사 |
| **배포 방식** | Google Workspace Admin Console에서 조직 단위(OU)별 ExtensionInstallForcelist 적용 |
| **목표** | 25,000 좌석, ARR $5M+, Extension 설치율 95%+ |
| **전략** | 계열사별 전담 CSM 배정, Chrome 관리 정책 표준화, 사내 IT 부서와 공동 배포 계획 수립 |
| **성공 기준** | 계열사 10개 이상 도입, NPS 50+, 자동화율 50%+, 일일 Active Extension 15,000+ |

**Stage 3: 외부 B2B (18~30개월) -- 고객사 Chrome 관리 정책 연동**

| 항목 | 내용 |
|------|------|
| **타겟** | 제조/금융/물류 대기업 (삼성, SK, LG 등) |
| **배포 방식** | 고객사 Google Workspace 또는 Chrome Browser Cloud Management 연동 배포 |
| **목표** | 50,000+ 좌석, ARR $12M+ |
| **전략** | 백서 발간, 컨퍼런스 발표, 레퍼런스 사례 마케팅, Chrome Extension 무설치 배포 차별화 강조 |
| **성공 기준** | 외부 고객 3개사 이상 계약 체결, Chrome Web Store 평점 4.5+ |

### 7.3 Chrome Extension 차별화 요소

| 요소 | 기존 접근 (앱/PWA) | Chrome Extension |
|------|-------------------|------------------|
| **설치** | 앱 스토어 다운로드 또는 URL 접속 필요 | ExtensionInstallForcelist로 자동 설치, 사용자 동작 불필요 |
| **컨텍스트** | 별도 앱 전환 후 수동 입력 | 현재 브라우징 페이지 컨텍스트 자동 인식 |
| **접근성** | 별도 앱/탭 열기 | Side Panel에서 즉시 접근, 작업 중단 없음 |
| **관리** | 개별 설치/업데이트 관리 | Chrome Admin Console 중앙 관리, 정책 기반 배포/제거 |
| **업데이트** | 수동 업데이트 또는 앱 스토어 의존 | Chrome 자동 업데이트로 항상 최신 버전 보장 |
| **제거 방지** | 불가능 | Enterprise 정책으로 사용자 제거 차단 가능 |

### 7.4 네트워크 효과

- **데이터 플라이휠**: Extension 사용자 증가 -> 브라우징 컨텍스트 데이터 축적 -> AI 품질 향상 -> 사용자 만족도 증가
- **에이전트 마켓**: 부서별 커스텀 에이전트 공유 -> 생태계 잠금 효과
- **통합 깊이**: 사내 웹 시스템 연동이 깊어질수록 교체 비용(Switching Cost) 기하급수적 증가
- **관리 정책 잠금**: Chrome 관리 정책에 Extension이 포함되면 IT 인프라 레벨 의존성 발생

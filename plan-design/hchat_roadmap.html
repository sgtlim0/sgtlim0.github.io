STRATEGIC INSIGHTS
핵심 발견사항

✦ 전략적 강점

1
Chrome Extension Only 단일화는 옳은 결정. 설치 장벽 제로 + Content Script 직접 DOM 접근 + 사용자 세션 공유라는 3가지 이점을 경쟁사는 구조적으로 따라올 수 없음.
2
4-Layer 수직 통합이 경쟁사와의 가장 큰 차별점. Perplexity/Arc/eesel 모두 이 중 1-2개 레이어만 보유. 전 레이어 통합은 18개월+ 진입 장벽.
3
Zero Trust + 한국 AI 기본법 대응이 엔터프라이즈 구매 결정의 핵심. 기업 68%가 데이터 보안을 최대 장벽으로 꼽는 시장에서 구조적 해결책 보유.
4
rtrvr.ai 81.39% 성공률 $0.12/작업 벤치마크는 Smart DOM 전략의 기술적 타당성을 외부에서 검증. Skyvern 대비 25배 비용 우위.

⚠ 핵심 리스크

1
LLM API 비용 과소 추정 (CRITICAL) MARS 7단계 × 일 100건 = 일당 최대 $140. $45K 예산이 약 320일 만에 소진. Fast Ops(Grok $50/1k tok) 우선 라우팅 전략 즉시 필요.
2
SPA/Shadow DOM 처리 미검증 SAP Fiori, Confluence React 앱은 동적 DOM이라 Content Script 접근이 막힐 수 있음. Sprint 0 Day 1에 PoC 필수.
3
hchat-pwa와의 관계 미정의 5건 기획서에서 Extension이 PWA를 대체하는지 병렬 운영인지 불명확. Sprint 0 착수 전 확정 필요.
4
ARR $12.1M (24개월) 낙관적 외부 B2B 포함 수치. 현대차그룹 내부만으로는 $5.64M(18개월)이 현실적. 외부 확장은 36개월+ 보수적 계획 권장.

◈ Extension 개발 주의사항

1
waitForDOMStable() 단일 resolve 보장 경쟁 Extension(Prism) 분석에서 다중 resolve 레이스 컨디션(CRIT-01)이 가장 빈번한 데이터 부패 원인. AbortController 패턴 필수.
2
API 키 Two-phase 삭제 신규 키 검증 성공 후에만 기존 키 삭제. 프로바이더 전환 중 사용자 잠금(CRIT-05) 방지.
3
getBoundingClientRect() 배치 읽기 RQFP 알고리즘의 DOM 순회 시 강제 reflow 방지(HIGH-03). 먼저 모든 크기 읽기 후 처리.
4
LLMQueue SSE 스트림 명시적 abort 탭 닫힘·새 요청 발생 시 이전 스트림 취소(HIGH-08). 메모리/소켓 누수 차단.

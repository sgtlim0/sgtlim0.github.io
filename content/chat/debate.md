# 크로스 모델 토론

크로스 모델 토론은 H Chat v3의 신규 기능으로, 여러 AI 모델이 서로의 답변을 비평하고 종합하여 최적의 결론을 도출하는 3라운드 토론 시스템입니다.

## 3라운드 토론 엔진

토론은 초기 답변 → 상호 비평 → 종합의 3단계로 진행됩니다.

### Round 1: 초기 답변

모든 선택된 모델이 질문에 대해 독립적으로 답변합니다. 각 모델은 다른 모델의 답변을 보지 않은 상태에서 자신의 관점을 제시합니다.

**목표**: 다양한 초기 관점과 접근 방식 수집

```
예시 질문: "원격 근무의 장단점은 무엇인가요?"

Claude: [생산성과 워크라이프 밸런스 중심 답변]
GPT-4: [팀 협업과 기업 문화 중심 답변]
Gemini: [기술적 인프라와 보안 중심 답변]
```

### Round 2: 상호 비평

각 모델이 다른 모델들의 답변을 읽고 비평합니다. 빠진 관점, 논리적 오류, 개선점 등을 지적하며 건설적인 피드백을 제공합니다.

**목표**: 답변의 약점 파악 및 다각도 검증

```
Claude → GPT-4/Gemini 비평:
"GPT-4는 협업을 강조했지만 비동기 협업 도구를 간과했고,
Gemini는 보안을 언급했지만 실질적인 솔루션이 부족합니다."

GPT-4 → Claude/Gemini 비평:
"Claude의 워크라이프 밸런스 논의는 좋지만 개인차를 고려하지 않았고,
Gemini는 기술에 치중해 인간적 측면을 놓쳤습니다."

Gemini → Claude/GPT-4 비평:
"Claude와 GPT-4 모두 정성적 분석은 훌륭하지만
데이터 기반 근거가 부족합니다."
```

### Round 3: 종합 및 합의

종합 모델(또는 지정된 모델)이 모든 답변과 비평을 검토하여 최종 결론을 정리합니다. 각 모델의 강점을 통합하고 약점을 보완하여 가장 균형 잡힌 답변을 제시합니다.

**목표**: 최적의 종합 답변 도출

```
종합 답변:
"원격 근무는 다음과 같은 장단점이 있습니다:

장점:
- 워크라이프 밸런스 개선 (Claude 지적)
- 비동기 협업 도구로 유연성 확보 (GPT-4 비평 반영)
- 클라우드 보안 솔루션으로 데이터 보호 (Gemini 관점 + GPT 보완)

단점:
- 개인별 적응도 차이 (GPT-4 비평 반영)
- 기업 문화 유지 어려움 (GPT-4 관점)
- 초기 인프라 투자 필요 (Gemini 관점)

[연구 데이터 기반 통계 추가] (Gemini 비평 반영)"
```

## 토론 기록 시각화

토론 진행 과정을 시각적으로 표시합니다:

- **라운드별 타임라인**: 각 라운드의 진행 상황 확인
- **모델별 답변 카드**: 각 모델의 기여를 명확히 구분
- **비평 관계도**: 어떤 모델이 누구를 비평했는지 연결선으로 표시
- **종합 하이라이트**: 최종 답변에서 각 모델의 기여도 색상 코딩

## 최종 합의 도출

3라운드가 완료되면 다음 정보가 제공됩니다:

- **종합 답변**: 모든 관점을 통합한 최종 결론
- **핵심 인사이트**: 토론을 통해 발견된 중요 포인트
- **모델별 기여도**: 각 모델이 최종 답변에 기여한 부분
- **합의 수준**: 모델 간 의견 일치도 (높음/중간/낮음)

## 구현 세부사항

### 핵심 파일

- `src/lib/debate.ts`: 토론 엔진 로직
  - `initDebate()`: 토론 세션 초기화
  - `executeRound1()`: 초기 답변 수집
  - `executeRound2()`: 상호 비평 수행
  - `executeRound3()`: 종합 답변 생성

- `src/components/DebateView.tsx`: 토론 UI 컴포넌트
  - 라운드별 진행 상황 표시
  - 모델별 답변 렌더링
  - 비평 관계 시각화
  - 종합 답변 하이라이팅

## 사용 사례

### 1. 복잡한 의사결정

```
"신규 사업 진출 시 고려해야 할 요소는 무엇인가요?"
→ 재무, 마케팅, 법률, 기술 등 다각도 분석
→ 각 모델의 전문 영역 활용
→ 포괄적인 의사결정 가이드 획득
```

### 2. 논쟁적 주제 탐구

```
"AI 규제는 어느 수준이 적절한가요?"
→ 각 모델의 서로 다른 관점 제시
→ 상호 비평으로 논리적 허점 보완
→ 균형 잡힌 결론 도출
```

### 3. 기술적 설계 검토

```
"마이크로서비스 vs 모놀리식 아키텍처, 어느 것을 선택해야 하나요?"
→ 각 아키텍처의 장단점 심층 분석
→ 실제 적용 사례와 함정 공유
→ 프로젝트 맥락에 맞는 최종 권장사항
```

## 사용 방법

1. 채팅 입력창에서 "토론" 모드를 선택합니다
2. 토론에 참여할 AI 모델 3개 이상을 선택합니다
3. 토론 주제나 질문을 입력합니다
4. 3라운드 토론이 자동으로 진행됩니다:
   - Round 1이 완료되면 Round 2로 자동 진행
   - Round 2가 완료되면 Round 3로 자동 진행
5. 최종 종합 답변과 토론 전체 기록을 확인합니다

## 팁

- **모델 선택**: 서로 다른 강점을 가진 모델을 조합하면 더 풍부한 토론이 가능합니다
- **질문 구성**: 명확하고 구체적인 질문일수록 심도 있는 토론이 진행됩니다
- **시간 고려**: 3라운드 진행에 일반 채팅보다 3-5배 긴 시간이 소요됩니다
- **결과 활용**: 종합 답변뿐만 아니라 각 라운드의 논의 과정도 유용한 인사이트를 제공합니다

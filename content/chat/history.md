# 대화 기록 관리

H Chat v2에서 도입된 대화 기록 관리 시스템은 모든 AI 대화를 체계적으로 저장하고 관리할 수 있는 강력한 기능을 제공합니다.

## 대화 저장 구조

모든 대화는 브라우저의 로컬 스토리지(`chrome.storage.local`)에 저장됩니다.

### 스토리지 키

| 키 | 데이터 타입 | 설명 |
|---|-----------|-----|
| `hchat:conversations` | `ConvIndex[]` | 대화 인덱스 (최근 200개) |
| `hchat:conv:{id}` | `Conversation` | 개별 대화 + 전체 메시지 |

### 대화 데이터 구조

```typescript
interface Conversation {
  id: string               // 고유 UUID
  title: string           // 대화 제목
  model: string           // 사용된 AI 모델
  provider: string        // AI 프로바이더 (bedrock/openai/gemini)
  createdAt: number       // 생성 타임스탬프
  updatedAt: number       // 최종 수정 타임스탬프
  messages: Message[]     // 메시지 배열
  tags?: string[]         // 태그 목록
  pinned?: boolean        // 고정 여부
  forkedFrom?: string     // 포크된 원본 대화 ID
}
```

## 대화 검색

HistoryView 탭에서 전체 대화 기록을 검색할 수 있습니다.

### 검색 기능

- **전체 텍스트 검색**: 대화 제목과 메시지 내용 모두 검색
- **실시간 필터링**: 입력하는 즉시 결과 표시
- **퍼지 매칭**: 비슷한 단어도 검색 결과에 포함
- **키보드 단축키**: `Ctrl+K` (전역 검색 모달 열기)

### 검색 사용 방법

1. HistoryView 탭 상단의 검색창 사용
2. 또는 `Ctrl+K` 단축키로 전역 검색 모달 열기
3. 검색어 입력 (한글/영문 모두 지원)
4. 검색 결과 클릭하여 해당 대화로 바로 이동

```typescript
// 검색 API 예시
import { searchMessages } from '../lib/messageSearch'

const results = await searchMessages('AWS 설정')
// 결과: { convId, message, snippet, matchScore }[]
```

## 태그 관리

대화를 분류하고 그룹화하기 위한 태그 시스템입니다.

### 태그 기능

- **자유 태그**: 원하는 태그 이름 자유롭게 생성
- **멀티 태그**: 하나의 대화에 여러 태그 지정 가능
- **태그 필터**: 특정 태그로 대화 목록 필터링
- **자동 완성**: 기존 태그 목록에서 선택 가능

### 태그 추가/삭제

```markdown
1. 대화 목록에서 대화 선택
2. 태그 아이콘(🏷️) 클릭
3. 드롭다운에서 기존 태그 선택 또는 새 태그 입력
4. Enter로 태그 추가
5. 태그의 X 버튼으로 삭제
```

### 권장 태그 예시

| 카테고리 | 태그 예시 |
|---------|----------|
| 업무 | `업무`, `회의`, `보고서`, `이메일` |
| 기술 | `코딩`, `디버깅`, `AWS`, `Python` |
| 학습 | `학습`, `튜토리얼`, `문서`, `개념` |
| 프로젝트 | `프로젝트명`, `클라이언트명` |

## 대화 고정 (Pin)

중요한 대화를 목록 최상단에 고정할 수 있습니다.

### 고정 기능

- **우선 표시**: 고정된 대화는 항상 목록 최상단에 표시
- **핀 아이콘**: 📌 아이콘으로 시각적 구분
- **무제한 고정**: 여러 대화 동시 고정 가능
- **고정 순서**: 고정된 시간 역순으로 정렬

### 고정 방법

```markdown
1. 대화 목록에서 대화 우클릭 또는 액션 버튼
2. "고정" 클릭
3. 대화가 목록 상단으로 이동
4. 다시 클릭하여 고정 해제
```

## 대화 포크 (Fork)

진행 중인 대화에서 새로운 분기를 만들 수 있습니다.

### 포크 개념

대화 포크는 특정 시점의 대화를 복사하여 새로운 대화를 시작하는 기능입니다. 원본 대화는 그대로 유지되며, 새 대화는 독립적으로 진행됩니다.

### 포크 인디케이터

- **포크 표시**: 포크된 대화는 `forkedFrom` 필드를 가짐
- **원본 링크**: 포크된 대화에서 원본 대화로 이동 가능
- **포크 트리**: 하나의 대화에서 여러 포크 생성 가능

### 포크 사용 시나리오

| 시나리오 | 설명 |
|---------|------|
| 대안 탐색 | 같은 질문에 다른 접근 방식 시도 |
| 모델 비교 | 같은 컨텍스트로 다른 AI 모델 테스트 |
| 분기 실험 | 대화 중간에서 다른 방향으로 전환 |
| 백업 | 중요한 시점의 대화 상태 보존 |

### 포크 방법

```markdown
1. ChatView에서 메시지 우클릭
2. "여기서 포크" 선택
3. 새 대화가 생성되며 해당 메시지까지의 히스토리 복사
4. 원본 대화는 변경되지 않음
```

## 가져오기/내보내기

대화 데이터를 백업하거나 다른 환경으로 이동할 수 있습니다.

### 내보내기 형식

H Chat은 5가지 내보내기 형식을 지원합니다.

| 형식 | 확장자 | 용도 |
|-----|-------|------|
| **Markdown** | `.md` | 문서화, 공유, GitHub |
| **HTML** | `.html` | 웹 페이지, 아카이브 |
| **JSON** | `.json` | 백업, 가져오기 (완전한 메타데이터) |
| **Plain Text** | `.txt` | 간단한 텍스트 보관 |
| **PDF** | `.pdf` | 인쇄, 보고서 (v3+) |

### 내보내기 방법

```markdown
1. ChatView 우측 상단 메뉴 버튼(⋮) 클릭
2. "내보내기" 선택
3. 원하는 형식 선택 (Markdown/HTML/JSON/TXT)
4. 파일 자동 다운로드
```

### JSON 내보내기 구조

```json
{
  "id": "conv-uuid",
  "title": "AWS Bedrock 설정 가이드",
  "model": "claude-sonnet-4-6",
  "provider": "bedrock",
  "createdAt": 1709294400000,
  "updatedAt": 1709298000000,
  "messages": [
    {
      "role": "user",
      "content": "AWS Bedrock 설정 방법 알려줘",
      "timestamp": 1709294400000
    },
    {
      "role": "assistant",
      "content": "AWS Bedrock 설정은 다음 단계로 진행됩니다...",
      "timestamp": 1709294405000
    }
  ],
  "tags": ["AWS", "설정"],
  "pinned": true
}
```

### 가져오기 방법

JSON 형식으로 내보낸 대화만 가져오기가 가능합니다.

```markdown
1. HistoryView 탭에서 "가져오기" 버튼 클릭
2. JSON 파일 선택
3. 메타데이터 검증 후 대화 복원
4. 중복 ID는 자동으로 새 UUID 생성
```

## 대화 삭제

더 이상 필요 없는 대화를 삭제할 수 있습니다.

### 삭제 기능

- **확인 다이얼로그**: 실수 방지를 위한 확인 절차
- **즉시 삭제**: 복구 불가능 (사전 백업 권장)
- **포크 관계 유지**: 포크된 대화가 있어도 원본 삭제 가능
- **대량 삭제**: 여러 대화 선택 후 일괄 삭제 (v3+)

### 삭제 방법

```markdown
1. 대화 목록에서 대화 우클릭 또는 액션 버튼
2. "삭제" 선택
3. 확인 다이얼로그에서 "삭제" 클릭
4. 대화가 영구 삭제됨
```

## 대화 보존 정책

### 자동 정리

- **인덱스 제한**: 최근 200개 대화만 인덱스에 유지
- **오래된 대화**: 자동으로 스토리지에서 제거 (수동 내보내기 권장)
- **사용량 기록**: 90일간 보존 (`usage.ts`)

### 스토리지 관리

Chrome 확장 프로그램의 스토리지 한계:
- `chrome.storage.local`: 최대 10MB (일반)
- `unlimitedStorage` 권한: 무제한 (H Chat은 사용 안 함)

대화가 많아지면 주기적으로 내보내기를 통해 백업하는 것을 권장합니다.

## 키보드 단축키

| 단축키 | 동작 |
|--------|------|
| `Ctrl+K` | 대화 검색 모달 열기 |
| `Ctrl+N` | 새 대화 시작 |
| `Ctrl+[` / `Ctrl+]` | 이전/다음 대화 |
| `Escape` | 모달 닫기 |

## 관련 파일

대화 기록 관리 시스템의 핵심 파일:

```
src/lib/chatHistory.ts      # 대화 CRUD + 인덱스
src/lib/messageSearch.ts     # 전체 텍스트 검색
src/lib/tags.ts              # 태그 관리
src/lib/exportChat.ts        # 4형식 내보내기
src/lib/importChat.ts        # JSON 가져오기
src/components/HistoryView.tsx  # UI 컴포넌트
```

## 참고

- 모든 대화 데이터는 로컬에 저장되며 외부로 전송되지 않습니다.
- 백업을 위해 정기적으로 JSON 형식으로 내보내기를 권장합니다.
- 포크 기능을 활용하면 대화의 여러 버전을 안전하게 실험할 수 있습니다.

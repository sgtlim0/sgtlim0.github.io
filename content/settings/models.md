# 모델 가이드

H Chat v3는 9개의 최신 AI 모델을 지원하며, 각 모델은 특정 작업에 최적화되어 있습니다.

## 지원 모델

### AWS Bedrock (Claude)

#### Claude Sonnet 4.6 (기본 권장)
- **모델 ID**: `us.anthropic.claude-sonnet-4-6`
- **용도**: 범용 작업, 일상적인 대화, 문서 작성
- **특징**:
  - 속도와 품질의 완벽한 균형
  - 대부분의 작업에 최적
  - 합리적인 비용 대비 성능
- **추천 상황**: 일반적인 질문, 글쓰기, 번역, 요약

#### Claude Opus 4.6
- **모델 ID**: `us.anthropic.claude-opus-4-6-v1`
- **용도**: 복잡한 분석, 깊은 추론, 장문 문서 처리
- **특징**:
  - 최고 수준의 추론 능력
  - 복잡한 멀티스텝 문제 해결
  - 긴 컨텍스트 이해력
- **추천 상황**: 연구 분석, 복잡한 코드 리뷰, 전략 수립

#### Claude Haiku 4.5
- **모델 ID**: `us.anthropic.claude-haiku-4-5-20251001-v1:0`
- **용도**: 빠른 응답이 필요한 간단한 작업
- **특징**:
  - 초고속 응답 속도
  - 낮은 비용
  - 간단한 작업에 충분한 성능
- **추천 상황**: 짧은 질문, 빠른 번역, 간단한 코드 설명

### OpenAI

#### GPT-4o
- **모델 ID**: `gpt-4o`
- **용도**: 코드 생성, 기술 작업, 프로그래밍 지원
- **특징**:
  - 코드 생성에 특화
  - 다양한 프로그래밍 언어 지원
  - 기술 문서 작성 우수
- **추천 상황**: 코드 작성, 디버깅, 기술 문서, API 설계

#### GPT-4o mini
- **모델 ID**: `gpt-4o-mini`
- **용도**: 빠르고 경제적인 일반 작업
- **특징**:
  - GPT-4o의 경량 버전
  - 빠른 응답 속도
  - 저렴한 비용
- **추천 상황**: 간단한 코드 질문, 빠른 검색, 일상 대화

### Google Gemini

#### Flash 2.0
- **모델 ID**: `gemini-2.0-flash-exp`
- **용도**: 초고속 응답, 실시간 대화
- **특징**:
  - 가장 빠른 응답 속도
  - 실험적 최신 기술 적용
  - 저지연 스트리밍
- **추천 상황**: 실시간 채팅, 빠른 정보 조회, 즉각적인 피드백

#### Pro 1.5
- **모델 ID**: `gemini-1.5-pro`
- **용도**: 긴 컨텍스트, 복잡한 작업
- **특징**:
  - 매우 긴 컨텍스트 윈도우 (최대 2M 토큰)
  - 대량의 문서 동시 처리
  - 복잡한 다단계 추론
- **추천 상황**: 장문 문서 분석, 여러 파일 비교, 코드베이스 전체 분석

## 자동 모델 라우팅

H Chat v3는 프롬프트의 특성을 분석하여 최적의 모델을 자동으로 선택합니다.

### 라우팅 로직

```
프롬프트 분석
    ↓
코드 관련? → YES → GPT-4o
    ↓ NO
    ↓
복잡한 추론? → YES → Claude Opus 4.6
    ↓ NO
    ↓
긴 문서? → YES → Gemini Pro 1.5
    ↓ NO
    ↓
빠른 응답? → YES → Claude Haiku 4.5 / Gemini Flash 2.0
    ↓ NO
    ↓
기본 선택 → Claude Sonnet 4.6
```

### 패턴 감지

#### 코드 관련 감지
프롬프트에 다음이 포함되면 GPT-4o 선택:
- 코드 블록 (```)
- 함수명 패턴 (`functionName()`)
- 프로그래밍 언어 키워드 (`const`, `def`, `class`, `import`)
- 파일 확장자 (`.js`, `.py`, `.ts`)

#### 복잡한 추론 감지
다음 키워드 포함 시 Claude Opus 4.6 선택:
- "분석", "비교", "평가", "전략", "설계"
- "왜", "어떻게", "이유"
- 다단계 질문 ("먼저...그 다음...")

#### 긴 문서 감지
다음 조건 시 Gemini Pro 1.5 선택:
- 프롬프트 길이 > 10,000자
- 여러 파일 내용 포함
- "전체", "모든", "모두" 키워드

#### 빠른 응답 감지
다음 조건 시 Haiku/Flash 선택:
- 프롬프트 길이 < 100자
- 단답형 질문 ("뭐", "누가", "언제", "어디")
- "빨리", "간단히", "짧게" 키워드

## 모델 선택 가이드

### 일반 사용자
- **시작**: Claude Sonnet 4.6 (기본 모델)
- **빠른 답변 필요**: Claude Haiku 4.5
- **중요한 분석**: Claude Opus 4.6

### 개발자
- **코드 작성**: GPT-4o
- **코드 리뷰**: Claude Opus 4.6
- **빠른 코드 설명**: GPT-4o mini
- **전체 코드베이스 분석**: Gemini Pro 1.5

### 연구자/분석가
- **문헌 분석**: Claude Opus 4.6
- **대량 문서 처리**: Gemini Pro 1.5
- **빠른 정보 조회**: Gemini Flash 2.0

### 비용 최적화
1. **일상적인 작업**: Claude Haiku 4.5 또는 GPT-4o mini
2. **중요한 작업만**: Claude Opus 4.6
3. **자동 라우팅 활용**: 비용과 성능의 자동 균형

## 수동 모델 선택

자동 라우팅을 원하지 않는 경우:
1. 채팅 입력창 옆의 모델 선택 드롭다운을 클릭합니다
2. 원하는 모델을 선택합니다
3. 선택한 모델이 해당 대화에 계속 사용됩니다

## 모델 전환 시 주의사항

- **컨텍스트 유지**: 같은 대화 내에서 모델을 전환해도 이전 대화 내용이 유지됩니다
- **스타일 차이**: 모델마다 응답 스타일이 다를 수 있습니다
- **토큰 제한**: 각 모델의 컨텍스트 윈도우를 확인하세요
- **비용 변화**: 모델마다 사용 비용이 다르므로 주의가 필요합니다
